# NFR-UX-001: Comprehension-first Interaction

The UI MUST prioritize comprehension over graph completeness. The active route MUST remain primary, while alternatives, terminal routes, and source citations SHOULD live in the detail panel.

## Acceptance Criteria

- Users can identify current scenario and current step without reading code or docs.
- Secondary options do not visually compete with the primary route.
- The detail panel explains what command is relevant, why, and what can happen next.
